## üìù Issue Guidelines

### Before Creating an Issue

1. **Search first**: Check if your issue already exists
2. **Use templates**: Select the appropriate issue template
3. **Be specific**: Provide clear, detailed information
4. **One issue per topic**: Don't combine multiple bugs or features

### Writing Good Issues

**Bug Reports** should include:
- Clear title describing the problem
- Steps to reproduce
- Expected vs actual behavior
- Code examples (DBML or Mermaid)
- Screenshots if applicable
- Environment (browser, OS)

**Feature Requests** should include:
- Problem statement
- Proposed solution
- Use cases
- Priority level

**Questions** should include:
- What you're trying to do
- What you've tried
- Code examples if relevant
